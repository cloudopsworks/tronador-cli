# Chocolatey packaging

Chocolatey packaging is generated by GoReleaser from `.goreleaser.yml`.
Do not use `internal/distribution` or `tools/distribution` generators.

The release pipeline uses the GoReleaser `publishers` section to call
`packaging/chocolatey/package.sh` for the Windows `amd64` archive. The script
builds a `.nupkg` directly from the GoReleaser artifact and does not require the
Chocolatey CLI for package generation. Publishing remains credential-gated: set
`TRONADOR_CHOCOLATEY_PUBLISH=true` and provide `CHOCOLATEY_API_KEY` only when CI
should push to the Chocolatey feed.

Primary validation:

```bash
goreleaser check
```
